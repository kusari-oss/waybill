# Contract: Gradle lockfile reader (US3, FR-005, FR-006)

**New module directory**: `mikebom-cli/src/scan_fs/package_db/gradle/`

Files:
- `mod.rs` (dispatcher + public entry point)
- `lockfile.rs` (handles BOTH `gradle.lockfile` and `buildscript-gradle.lockfile`)

## Trigger

Either file named `gradle.lockfile` OR `buildscript-gradle.lockfile` anywhere in the scan tree. Both files are parsed by the same code path; only the filename determines lifecycle scope.

## Parsing

Line-oriented via `std::str::Lines`. The Gradle lockfile format is well-documented and stable:

```
# This is a Gradle generated file for dependency locking.
# Manual edits can break the build and are not advised.
# This file is expected to be part of source control.
com.google.guava:guava:32.1.3-jre=compileClasspath,runtimeClasspath,testCompileClasspath,testRuntimeClasspath
org.jetbrains.kotlin:kotlin-stdlib:1.9.20=compileClasspath,runtimeClasspath
empty=annotationProcessor,kapt
```

Per-line parsing:
1. Skip lines starting with `#` (comments).
2. Skip the literal `empty=...` line (lists configurations with no resolved deps).
3. Split on `=` once: left side is `group:name:version`, right side is comma-separated configurations.
4. Split the left side on `:` into exactly 3 parts (group, name, version).
5. Malformed lines (wrong segment count, missing `=`, etc.) trigger `tracing::warn!` and skip.

## PURL derivation

```
pkg:maven/<group>/<name>@<version>
```

Group with dots (`com.google.guava`) is preserved as-is in the namespace segment — matches the existing `maven.rs` PURL emission.

## Lifecycle-scope mapping (FR-006)

| File | LifecycleScope | CDX `scope` | SPDX 2.3 relationship | SPDX 3 lifecycleScope |
|---|---|---|---|---|
| `gradle.lockfile` | `None` (runtime default) | absent (default = required) | regular `DEPENDS_ON` | absent |
| `buildscript-gradle.lockfile` | `Some(Build)` | `"excluded"` | `BUILD_DEPENDENCY_OF` (reversed direction per spec) | `lifecycleScope: "build"` |

The lifecycle-scope → CDX `scope` mapping is the existing milestone-052 path at `generate/cyclonedx/builder.rs:590-605`. No new code needed; just set `lifecycle_scope` on the `PackageDbEntry` and the existing emission code handles all three formats.

## Annotations emitted (per component)

| Annotation | Value |
|---|---|
| `mikebom:source-files` | absolute path of the lockfile (`gradle.lockfile` or `buildscript-gradle.lockfile`) |
| `mikebom:lifecycle-scope` | `"build"` for buildscript entries; absent for runtime |
| `mikebom:gradle-configurations` | comma-joined list of configurations (e.g. `"compileClasspath,runtimeClasspath"`) — informational only, downstream filterable |

## Edge cases handled

- **`empty=<configs>` line**: skipped (matches Gradle's own convention for "this configuration was resolved but has zero deps").
- **Multi-segment group with extra `:`** (rare but legal: `org.foo.bar:baz:1.0`): the segment count is `>= 3`, so the parser takes the last two as `name:version` and joins the rest as `group`. Robust to deep namespaces.
- **Header line**: `# Generated by Gradle...` is comment-stripped via `line.starts_with('#')`.
- **Trailing whitespace / blank lines**: trimmed; empty lines skipped.
