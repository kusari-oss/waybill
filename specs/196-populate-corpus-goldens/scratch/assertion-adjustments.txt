For each of the 4 non-cobra source targets (rust-ripgrep, npm-express,
python-flask, maven-guice), the m195-authored assertions expected
`graph-completeness == complete`. Empirical scans show `partial` with
various legitimate reasons. Per m195 research §R8, assertions
adjusted to match observed shape; tripwires preserved as documented in
each function's expanded doc-comment.

Also for each target, the m195-authored "main-module PURL present"
check assumed the ecosystem-native mainmod PURL. Under the harness's
`--root-name <target> --root-version <sha7>` invocation, m077 operator-
override DROPS the ecosystem mainmod and replaces the metadata.component
with `pkg:generic/<target>@<sha7>`. Same pattern go-cobra already
exhibits in m195. Assertions relaxed to check for "at least one
`pkg:<ecosystem>/*` transitive" — this preserves the "reader is
emitting anything" tripwire without spuriously firing on the
operator-override subject drop.

## Tripwires preserved

- `graph-completeness == "partial"` (each function): a regression that
  flips to `unknown` or `<missing>` (classifier bypass) still trips.
- `pkg:<ecosystem>/*` transitives present: a regression that breaks
  the reader entirely (drops all transitives) still trips.
- Individual per-format assertions (SPDX 2.3, SPDX 3) inherit from CDX
  via the same layer1 function signature (currently only CDX asserted;
  future work: cross-format parity assertions).

## Tripwires NOT preserved (documented explicitly)

- Per-count orphan tightening (e.g., "flask MUST have exactly 94 orphans"
  vs "≤ 100"). Not encoded — would generate churn on legitimate flask
  version bumps that add/remove a few dev deps. Layer 2 golden diff
  covers count-drift via byte-identity instead.
- Per-reason-code narrowing (e.g., "npm-express reason MUST be exactly
  `transitive-edges-unresolvable: npm`" vs any `partial`). Not encoded;
  future work: extend `AssertionFailure` diagnostics to include the
  observed reason for review-time inspection without hard-failing.
