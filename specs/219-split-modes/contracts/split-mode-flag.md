# Contract: `--split[=<mode>]` CLI flag

**Feature**: 219-split-modes | **Related**: FR-001, FR-008

## Surface

Change the type of `waybill_cli::cli::scan_cmd::ScanArgs.split` from `bool` to `Option<crate::generate::split::SplitMode>` where `SplitMode: clap::ValueEnum`.

### Accepted invocations

```
waybill sbom scan --split --output-dir out/            # → Some(Workspace) via default_missing_value
waybill sbom scan --split=workspace --output-dir out/  # → Some(Workspace) explicit
waybill sbom scan --split=directory --output-dir out/  # → Some(Directory)
waybill sbom scan --output-dir out/                    # → None (no split)
```

### Rejected invocations

- `waybill sbom scan --split=<nonexistent>` → clap parse error, non-zero exit, stderr message listing accepted values (per FR-008).
- `waybill sbom scan --split directory` (space-separated) → clap parse error via `require_equals = true`. Prevents `directory` from being silently consumed as the next positional arg.
- `waybill sbom scan --split=<empty>` → clap parse error (empty string isn't a valid variant).
- `waybill sbom scan --split=DIRECTORY` → clap parse error (uppercase; `rename_all = "lowercase"` normalizes only variant *rendering*, not accepted casing).
- `waybill sbom scan --split` without `--output-dir` → existing m215 error (`--split requires --output-dir`). Unchanged.
- `waybill sbom scan --split --output out.json` → existing m215 error (`conflicts_with = "output"`). Unchanged.

## Precedents

- **m173 `--warm-go-cache <MODE>`** at `scan_cmd.rs:1190-1198`: `value_enum`, `default_value = "off"`, `require_equals = true`, `num_args = 1`. Same clap pattern shape.
- **m207 `--offline[=<VALUE>]`** at `scan_cmd.rs:~192`: `require_equals = true` precedent + accepts `true`/`false` values. Documents WHY `require_equals` matters: keeps the next positional arg from being silently consumed.

The m219 flag combines both: enum-typed value + optional presence (bare = default).

## Clap attribute stack

```rust
#[arg(
    long,
    value_enum,
    num_args = 0..=1,
    default_missing_value = "workspace",
    require_equals = true,
    conflicts_with = "output"
)]
pub split: Option<crate::generate::split::SplitMode>,
```

Semantics:
- `long`: flag name = `--split` (auto-derived from field name).
- `value_enum`: use the `SplitMode: ValueEnum` derive for parsing/help.
- `num_args = 0..=1`: accepts both `--split` (0 args) and `--split=<value>` (1 arg).
- `default_missing_value = "workspace"`: bare `--split` resolves to the `workspace` variant.
- `require_equals = true`: reject `--split directory`; require `--split=directory`.
- `conflicts_with = "output"`: preserves m215 constraint.

## Help output shape (`waybill sbom scan --help`)

```
      --split [<SPLIT>]
          Milestone 215/219 — split monorepo SBOM into per-workspace-
          member or per-directory sub-SBOMs. ...
          [possible values: workspace, directory]
```

`[possible values: workspace, directory]` auto-generated by clap from the `ValueEnum` derive.

## Consumer behavior

At `run_scan` entry:

```rust
// Before (m215):
if args.split {
    let output_dir = args.output_dir.as_ref().expect("...");
    let handled = emit_split(..., ...); // no mode param
    ...
}

// After (m219):
if let Some(mode) = args.split {
    let output_dir = args.output_dir.as_ref().expect("...");
    let handled = emit_split(..., mode); // NEW mode param
    ...
}
```

No behavior change when `args.split == None` (no split; existing single-SBOM path).

## Backward compatibility

- Bare `--split` post-m219 IS the pre-m219 bare `--split` semantically (both → workspace mode).
- Callers of the shipped alpha.67 binary that pass bare `--split` see zero output change.
- Callers that omit `--split` entirely see zero output change.
- New callers passing `--split=directory` opt into the new behavior explicitly.
- SC-005 byte-identity gate enforced by extending m215's existing split test suite (no golden regen; all m215 tests MUST pass unchanged on the m219 branch).

## Failure modes

| Input | Behavior |
|---|---|
| `--split=nonexistent-mode` | Clap parse error; non-zero exit; stderr lists `workspace`, `directory`. |
| `--split=""` (empty value) | Clap parse error; non-zero exit. |
| `--split=WORKSPACE` | Clap parse error; non-zero exit. |
| `--split=directory` without `--output-dir` | Existing m215 error: `--split requires --output-dir`. Unchanged. |
| `--split=directory --output out.json` | Existing m215 error: `conflicts_with = "output"`. Unchanged. |
| `--split=directory --output-dir out/` on a scan with zero main-modules | m215 fallback: WARN log + single-SBOM emit (per FR-009). Unchanged. |
