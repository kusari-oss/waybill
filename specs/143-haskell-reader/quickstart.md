# Quickstart — milestone 143 Haskell reader

Operator-facing walkthrough of the scenarios this milestone surfaces.

## Scenario 1 — Scan a cabal-managed Haskell project (US1 / SC-001)

```bash
mikebom --offline sbom scan --path . --output /tmp/app.cdx.json
```

```bash
# Main-module (single-package layout):
jq '.metadata.component' /tmp/app.cdx.json
# {"bom-ref": "pkg:hackage/my-app@1.2.3",
#  "name": "my-app", "version": "1.2.3",
#  "purl": "pkg:hackage/my-app@1.2.3",
#  "properties": [
#    {"name": "mikebom:component-role", "value": "main-module"}
#  ]}

# Lockfile-derived deps:
jq '.components[] | select(.purl | startswith("pkg:hackage/")) | .purl' /tmp/app.cdx.json | sort | head -10
# "pkg:hackage/aeson@2.2.0.0"
# "pkg:hackage/attoparsec@0.14.4"
# "pkg:hackage/base@4.18.0.0"
# "pkg:hackage/bytestring@0.11.5.3"
# "pkg:hackage/containers@0.6.7"
# "pkg:hackage/text@2.0.2"
# ...
```

## Scenario 2 — Boot-library discrimination (Q1 / FR-014 / SC-011)

```bash
# Filter to GHC-shipped boot libs (Q1 informational annotation):
jq '.components[] | select(.properties[]? | .name == "mikebom:ghc-stdlib" and .value == "true") | .name' /tmp/app.cdx.json | sort
# "array"
# "base"
# "bytestring"
# "containers"
# "deepseq"
# "text"
# ...

# Filter to user-pulled deps only (operator-actionable discrimination):
jq '[.components[] | select(.purl | startswith("pkg:hackage/")) | select(.properties[]? | (.name != "mikebom:ghc-stdlib"))] | length' /tmp/app.cdx.json
```

## Scenario 5 — Multi-stanza `*.cabal` with per-stanza scope (Q2 / SC-012)

```bash
# A *.cabal declaring library + executable + test-suite + benchmark
mikebom --offline sbom scan --path my-multi-stanza-app --output /tmp/multi.cdx.json

# Filter test-only deps (hspec, criterion) via CDX scope=excluded:
jq '.components[] | select(.scope == "excluded") | .name' /tmp/multi.cdx.json
# "hspec"
# "criterion"

# --exclude-scope dev (top-level flag BEFORE sbom subcommand) suppresses:
mikebom --offline --exclude-scope dev sbom scan --path . --output /tmp/prod.cdx.json
jq '.components[] | select(.name == "hspec")' /tmp/prod.cdx.json
# (empty)
```

## Scenario 7 — Hpack detect-and-warn (Q3 / FR-015)

```bash
ls /tmp/hpack-project/
# package.yaml
# my-app.cabal     # generated by hpack

head -1 /tmp/hpack-project/my-app.cabal
# -- This file has been generated from package.yaml by hpack version 0.36.0.

mikebom --offline sbom scan --path /tmp/hpack-project --output /tmp/h.cdx.json 2>/tmp/scan.log

grep 'Hpack' /tmp/scan.log
# WARN ... haskell: Hpack-generated *.cabal detected alongside package.yaml — run 'hpack' to regenerate before scanning if package.yaml has been edited
```

## Cross-format byte-equivalence check

```bash
mikebom --offline sbom scan --path my_haskell_app \
  --format cyclonedx-json --format spdx-2.3-json --format spdx-3-json \
  --output cyclonedx-json=/tmp/h.cdx.json \
  --output spdx-2.3-json=/tmp/h.spdx.json \
  --output spdx-3-json=/tmp/h.spdx3.json

jq '[.components[] | select(.purl | startswith("pkg:hackage/")) | .purl] | sort' /tmp/h.cdx.json > /tmp/cdx-hk.txt
jq '[.packages[].externalRefs[]? | select(.referenceType == "purl") | .referenceLocator | select(startswith("pkg:hackage/"))] | sort' /tmp/h.spdx.json > /tmp/spdx-hk.txt
jq '[.["@graph"][] | select(.software_packageUrl? | tostring | startswith("pkg:hackage/")) | .software_packageUrl] | sort' /tmp/h.spdx3.json > /tmp/spdx3-hk.txt

diff /tmp/cdx-hk.txt /tmp/spdx-hk.txt
diff /tmp/cdx-hk.txt /tmp/spdx3-hk.txt
# (no output = success)
```

## Known deferrals (spec Out of Scope)

- Hpack `package.yaml` direct parsing (v1 detects + warns only).
- cabal-install `plan.json` (v1 reads `cabal.project.freeze` only).
- Pre-cabal-2.4 `cabal.config` legacy freeze format.
- Pre-Stack-2.1 lockfile formats.
- Full Stackage snapshot expansion (~2500 packages per snapshot — would require network).
- Custom Hackage mirrors.
- Per-package transitive dep edges.
- License extraction from `*.cabal` `license:` field (v1.1).
- Stack `stack.yaml.lock` git-source `extra-deps` (v1.1; warn-and-skip in v1).
- Live `cabal` / `stack` / GHC invocation.
